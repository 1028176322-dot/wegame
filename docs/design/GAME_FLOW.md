## 1. 总览

玩家旅程一句话：**打开小游戏 → 被引导着建起第一座塔 → 在一局里"养"出清屏的爽感 → 局间用产出解锁/变强 → 每天回来看排行/签到/赛季，越养越上头。**

本文把旅程拆成 7 个阶段，每个阶段标注玩家*感受到什么*、*在做什么决策*、对应功能 ID 与 GDD 章节。所有数值为 `[PLACEHOLDER]`（见 GDD §6），流程不含魔法数字。

## 2. 阶段拆分（玩家时间线）

### 2.1 冷启动与首进（First Launch）
- **发生**：微信内点击进入；引擎加载主包 + 分包资源（F30）；`onShow` 拉起（F31）。
- **玩家感受**：快、不卡、无多余弹窗。
- **决策**：无（系统层）。
- **分支**：
  - 有本地存档（F11 / F32）→ 直接进入大厅（2.5）。
  - 无存档（新玩家）→ 走新手引导（2.2）。
- **合规**：首次进入按微信规范处理授权 / 隐私提示（F39 设置入口可见）。

### 2.2 新手引导（Onboarding，F10，P3 零门槛）
按 GDD §7 清单，保证首波保送、无失败可能：
1. 进入一个低压教学局（固定短图）。
2. **0–30s 引入核心动词**：点塔位建箭塔 → 即时首杀飘字（F7 反馈）。
3. 引导建风塔（F3 招牌），展示击退控制（第二类控制，与冰塔 combo）。
4. 引入"应急兑换"（F5）与"养塔"（F4）：把木喂给箭塔，看它变强（木主源为怪掉）。
5. **至少一个机制靠探索发现**（非文本）：如地图边缘的风塔位不加文字提示，让玩家自己点，看它击退怪物。
6. 结束于钩子：首局胜利 → 弹出"解锁炮塔 / 再来一局"（2.5）。
- **成功标准**：无设计师辅助下，新玩家首局理解"建 → 养 → 清屏"，且主动建起风塔。

### 2.3 单局核心循环（Session Core Loop）
这是 Fun Hypothesis 的载体：**看自己养的塔从弱到一发清屏。**

- **Moment-to-Moment (0–30s)**
  - 动作：点空塔位建塔（F2，单指）。
  - 反馈：建塔音效 + 塔亮起；首发命中即飘字伤害（F7）。
  - 奖励：首杀的金币（F5）+ 成就感。
- **Session Loop (3–5min)**
  - 目标：撑过本局全部波次（F6），Boss 波前做"应急兑换养塔"取舍。
  - 张力：金用于建、木用于养（F5 双币压力）；风塔（F3）击退控制但占塔位。
  - 结构：普通波 → 护甲 / 克制波（迫使调整塔种） → Boss 波（高压，检验养成）。
  - 风险：漏怪扣 Lives（F8），0 命即失败。
  - 操控：HUD（F9）显示波次 / 命 / 金木 / 2x / 暂停；支持加速与单指。
- **局内循环**：每一波 = 观察威胁 → 决定建 / 养 / 换 → 看结果 → 下一波（波次循环 ↻）。

### 2.4 局间结算与产出（F12 / F19）
- **胜**：按波数 / 漏怪数结算金币（木为 session 不结算，仅本局养塔用）；推进解锁进度（F13）。
- **负**：失败演出（F40 暂不做）→ 简版结算 → 鼓励"再来一局"。
- **广告 / 分享（F19，带代码开关 default off）**：开关开启时，看广告 / 分享可额外得木或复活；开关 off 时经济**无任何额外产出**（防破环）。
- 产出进入元进度资源池（2.5）。

### 2.5 元进度与大厅（Meta / Hub，F13–F18）
局间停留的大厅 / 主界面：
- 解锁塔种（F3 解锁制）：达到玩家等级门槛（S29 `unlock_config.required_level`）解锁——首发箭/炮/冰/风无门槛；魔 L[PLACEHOLDER] / 毒 L[PLACEHOLDER] / 电 L[PLACEHOLDER]（具体阈值见后续数值设计表 / S29）。
- 永久升级（F13）：投资某条成长线。
- 每日签到（F14）：回归钩子。
- 成就（F37）/ 图鉴（F38）：目标感与查询。
- 异步排行榜（F15）：被他人进度推动。
- 设置（F39）：音量 / 震动 / 字号。

### 2.6 长线留存与社交（Long-term，F14 / F15 / F19 / F21）
- **每日回归**：签到 + 排行榜名次变化 + （探索）赛季 F21。
- **社交传播**：分享（F19 开关）得奖励，分享战绩图。
- **防作弊（F43）**：保护排行与经济不被改包破坏。
- **数据分析（F44）**：监控通胀（GDD §6 阈值），触发平衡 pass。
- **远程配置（F33）**：不改包热更波数 / 掉率 / 兑换率。

### 2.7 退出 / 中断 / 反馈
- 切后台 `onHide` 自动暂停（F31）；回前台 `onShow` 恢复。
- 失败 / 退出无惩罚（原型可逆性）。
- 客服 / 反馈入口（F41）常驻。

## 3. 状态机视角（关键状态）

```
Launch
  └─ 有存档? ── 是 ─→ Hub(2.5)
         └─ 否 ─→ Onboarding(2.2) ─→ InSession
InSession(循环): Build → Wave → Settle
  ├─ Lives>0 且波次未完 → 下一波（波次循环 ↻）
  ├─ 全部波次过 → Settle(胜) → Hub
  └─ Lives==0 → Settle(负) → Hub
Hub ── 每日/排行/赛季 ──→ 下一局 InSession
异常: onHide → Paused → onShow → Resume
```

## 4. 开放变量（仍待定，影响流程）
- 单局波数（普通波 / Boss 波配比）。
- 元进度资源来源（是否含广告 / 分享，与 F19 开关耦合）。
- 每日签到 / 排行榜刷新节奏。
- 失败演出预算（F40 暂不做）。

## 5. 与设计支柱 & Feature Scope 对应

| 设计支柱 | 流程落点 |
|---------|---------|
| 循环即策略 | 2.3 波次循环、护甲克制 |
| 养成爽感 | 2.2 / 2.3 养塔、2.5 解锁 |
| 一指可玩 | 2.1–2.3 单指 / HUD |
| 每局有取舍 | 2.3 应急兑换养塔、风塔占塔位 |
| 短局长线 | 2.3 短局 + 2.5 / 2.6 长线 |
