## 0. 一句话定位

在一条**环形赛道**上建塔、用**金币→木头**的经济循环把单塔"养"到变态强，看怪物一圈圈被清空——**一指操作、三分钟一局、跨局变强**。

---

## 1. 设计支柱（Design Pillars）

> 任何后续设计决策都须逐条对照这 5 条；违例须显式记录在 ADR。

| # | 支柱 | 不可妥协的玩家体验 | 反面信号（出现即预警） |
|---|---|---|---|
| P1 | **循环即策略** | 环形路径 + 金木经济循环是核心玩法，不是装饰 | 玩家只堆塔不碰经济也能通关 |
| P2 | **养成爽感** | 单塔深度养成（"养塔"）带来指数级成长的正反馈 | 所有塔同级、无成长曲线 |
| P3 | **一指可玩** | 触屏单指完成建造/升级/出售全部操作，零学习门槛 | 需要多指手势或二级菜单才能推进 |
| P4 | **每局有取舍** | 资源有限，"先经济还是先战力"是真实决策 | 最优解唯一、无 build 多样性 |
| P5 | **短局长线** | 3 分钟一局爽完即走；跨局解锁 + 排行榜构成留存 | 单局 >8 分钟或毫无元进度 |

## 2. Fun Hypothesis（乐趣假设）

> **唯一必须感觉对的爽点**：看着自己"养"起来的那座塔，从一下打不死小怪，到一发清屏。
> 若这一条不成立，其余系统都不重要。试玩优先验证它。

辅助爽点（次要）：环形地图的"包围感"、漏怪瞬间的紧张、Boss 波的极限换塔。

## 3. 核心循环（Core Loop）

### Moment-to-Moment（0–30 秒）
- **Action**：玩家点一个空格子 → 弹出塔种轮盘 → 选塔建造；或点已有塔 → 升级/出售。
- **Feedback**：建造有落位音 + 网格高亮；升级有光效 + 数值跳动；攻击有命中顿帧。
- **Reward**：即时——塔就位即可开火，第一只怪倒下给金币 + 屏幕飘字。

### Session Loop（3–5 分钟 / 一局）
- **Goal**：撑过本局所有波次（默认 `[PLACEHOLDER]` 50 波），Boss 波不漏或漏在容错内。
- **Tension**：金币有限 → 先靠怪掉木养核心塔 / 应急兑换，还是先补塔防漏？Boss 波前必须凑齐对应护甲克制。
- **Resolution**：通关 → 结算金币/元进度/漏怪数 → 解锁或强化元进度（木为 session 不结算；**玩家等级随 XP 提升，见 §5.9，达等级门槛解锁新塔种/功能**）；失败（Lives 归零）→ 本局结束，保留跨局资源。

### Long-Term Loop（小时–周）
- **Progression**：跨局元进度——解锁新塔种、永久强化（塔基伤/起始资源/漏怪容错）。
- **Retention Hook**：每日首通奖励（可变奖励日程）、好友排行榜（微信关系链）、赛季主题塔。

## 4. 地图与塔位模型

### 4.1 环形路径（"循环圈"）
- 怪物从**入口**进入，沿一条闭合环线行进，绕场 N 圈后抵达**终点出口**（参考绿圈：怪是"绕圈"而非直线到头）。
- 设计含义：塔放在环**内侧**，怪物每圈都经过射程 → 多圈累计伤害，养成塔越养越值。
- `[PLACEHOLDER]` 圈数、环线长度、转弯数（影响 AOE 塔的覆盖效率）。

### 4.2 塔位（插槽）
- 路径内侧为离散**格子/插槽**，分**内环 / 外环**两档：外环射程覆盖更多路段但更贵，内环便宜但覆盖短。
- 触屏交互：点空格 → 塔种轮盘（仅显示已解锁塔种）；点已有塔 → 操作卡（升级 / 出售 / 锁定目标模式）。
- 约束：每格仅 1 塔；出售返还 `[PLACEHOLDER]` 50% 投入。

## 5. 机制规格（Mechanic Specs）

### 5.1 Mechanic: 建造（Build）
- **Purpose**：把玩家意图落到地图。
- **Player Fantasy**："我要在这里架一座炮。"
- **Input**：点空格子 → 选塔种。
- **Output**：扣金币，格子生成塔，进入战斗；触发首杀前的"蓄势"微动画。
- **Success Condition**：塔就位且能命中进入射程的怪。
- **Failure State**：金币不足无法建造（按钮置灰，提示差额）。
- **Edge Cases**：格子被占、塔种未解锁、连点重复建造（防抖）。
- **Tuning Levers**：各塔造价、格子总数、内外环价差。
- **Dependencies**：Economy（金币）、Waves（怪物进入）。

### 5.2 Mechanic: 养成升级（养 / Upgrade）
- **Purpose**：P2 养成爽感的核心载体。
- **Player Fantasy**："我把这座塔喂到全场最强。"
- **Input**：点塔 → 消耗 **session 木头**升级（木来自怪物掉落 S03/S04 + 应急兑换；**不持久化、每局重来**，开局恒 Lv1）。可连续点，等级无硬上限软封顶 `[PLACEHOLDER]`。
- **Output**：塔等级 +1 → 伤害/攻速/特殊效果按曲线（沿用 `growth` 系数）提升；飘字 + 升阶光效；达阈值解锁被动技（见 §5.8）。
- **Success Condition**：高等级塔在 Boss 波能 solo 清场（验证 P2）。
- **Failure State**：木头耗尽卡在中段（提示"去刷木 / 应急兑换"）。
- **Edge Cases**：满级再点（置灰）、木头为负（禁止）、出售已养塔（等级归 Lv1，被动重新锁定）。
- **Tuning Levers**：每级木头消耗曲线、每级伤害增益、软封顶等级、被动解锁等级。
- **Dependencies**：Economy（session 木头）、Build、Skill System（§5.8）。

### 5.3 Mechanic: 双货币经济（Economy: 金币 / 木头）
- **Purpose**：P1 + P4——经济循环是策略本体（**局内双币**：金建 / 木养）。
- **金币 Gold（session）**：击杀怪掉落 + Boss 奖励 + 通关结算。用于**建造**与**应急兑换木**。局结束结算为元进度（见 §5.7 胜负 / S11）。
- **木头 Wood（session，不可带出副本、不持久化）**：用于**养塔**升级。来源 = **怪物概率掉落（主源，S04 `drop_wood_*`）+ 受限"应急兑换"（金→木，S03）**。~~原"木房"塔被动产木 / 通关木奖励已移除~~。
- **应急兑换率**：`1 木 = [PLACEHOLDER] 金`（**较差**汇率，受次数/触发条件限，是金→木唯一路径，详见 §6）。
- **Player Fantasy**："先看怪掉木节奏，再决定养哪座塔；急了才点应急兑换。"
- **Input**：怪物死亡自动掉木（飘字）；点或满足触发条件时"应急兑换"按钮（批量可选，受次数/条件限）。
- **Output**：金币↓木头↑（仅应急时）；木仅局内消耗于养塔。
- **Success Condition**：玩家靠"怪掉木→养塔"在 Boss 前养起核心塔且不崩盘。
- **Failure State**：经济崩——应急兑换太频繁导致金荒战力不足漏怪；或囤木太久被高波碾过。
- **Edge Cases**：金币为 0 无法应急兑换、应急次数耗尽、木超 session cap 截断。
- **Tuning Levers**：应急汇率、应急次数/触发阈值、掉木率/掉木量、击杀金币。
- **Dependencies**：Build、Upgrade、Waves、Skill System（§5.8）、所有战斗结算、Save（仅元进度持久化）。

### 5.4 Mechanic: 漏怪与生命（Leak & Lives）
- **Purpose**：失败压力与风险/回报的张弛。
- **Player Fantasy**："这波能漏两只吗？省下木头养塔。"
- **Input**：怪抵达终点出口。
- **Output**：按怪类型扣 Lives（小怪 -1，Boss -`[PLACEHOLDER]` 5，魔免怪同小怪）。
- **Success Condition**：Lives > 0 撑到通关。
- **Failure State**：Lives 归零 → 本局失败。
- **Edge Cases**：同一帧多怪抵达（累加扣）、Boss 漏怪的连锁惩罚、最后一波刚好归零的边界。
- **Tuning Levers**：起始 Lives `[PLACEHOLDER]` 20、各类怪漏怪惩罚、容错（元进度可 +Lives）。
- **Dependencies**：Waves、Economy（失败保留跨局资源）。

### 5.5 Mechanic: 波次与怪物（Waves）
- **Purpose**：难度曲线与护甲克制（P4 决策来源）。
- **Wave 结构**：常规波（递增 HP）+ 每 `[PLACEHOLDER]` 10 波 Boss（高 HP / 特殊护甲）+ 类型波（空军需对空塔、魔免需物理塔、重甲需魔法塔）。
- **Player Fantasy**："这波是魔免，我得提前架物理塔。"
- **Input**：计时/手动开下一波（默认自动，可调手动加速）。
- **Output**：刷怪、结算金币、触发 Boss 演出。
- **Success Condition**：玩家在类型波前完成对应塔种布局。
- **Failure State**：克制错配 → 大面积漏怪。
- **Edge Cases**：加速波导致资源来不及、Boss 与类型波重叠、空地同时来空/地怪。
- **Tuning Levers**：波数、HP 曲线、Boss 频率、类型分布。
- **Dependencies**：全部战斗机制、Economy、Leak。

### 5.6 塔种原型（Tower Archetypes）
> 每种对应一类护甲克制，保证 P4（无唯一解）。具体数值 `[PLACEHOLDER]`。

| 塔种 | 伤害类型 | 克制 | 特色 | 造价倾向 |
|---|---|---|---|---|
| 箭塔 | 物理单攻 | 轻甲 | 高频低伤，便宜 | 低 |
| 炮塔 | 物理溅射 | 集群/重甲 | AOE，外环优 | 中 |
| 魔法塔 | 魔法（无视护甲） | 重甲 / 高HP肉盾（无视护甲真伤）；**魔免怪免疫魔法(×0)，弱物理塔** | 稳定真伤 | 中 |
| 冰塔 | 控制（减速） | 所有 | 不输出，保命 | 低 |
| 毒塔 | 持续 DoT | 高 HP | 越肉越赚 | 中 |
| 电塔 | 连锁 | 密集 | 多目标弹射 | 高 |
| 风塔 | 控制（击退/位移） | 所有 | 击退怪物延长在场时间，与冰塔 combo | 中 |

> 塔 roster 共 7：**首发 4 = 箭 / 炮 / 冰 / 风塔**（风塔为**新风种，DO 已确认**命名/定位，替换原木房）；**解锁 3 = 魔 / 毒 / 电**（经 S11 元进度）。每塔种均含 1 主动 + 2 被动技能，详见 §5.8 / S28。

### 5.7 Mechanic: 胜负（Win / Lose）
- **Win**：撑过全部波次且 Lives > 0 → 结算 + 元进度。
- **Lose**：Lives = 0 → 本局结束，保留跨局资源（失败不惩罚，降低挫败，符合 P5 短局）。

### 5.8 Mechanic: 技能系统（Skill System，新增）
> 承载 DO 新规"每塔种 1 主动 + 2 被动"。详细规格见 **S28 技能系统**；本条为 GDD 层权威定义。
- **Purpose**：放大养成爽感（P2）与每局取舍（P4）——技能是塔"性格"的延伸，非新资源体系。
- **每塔种三技能（均不同）**：1 主动技 + 2 被动技。主动技 CD 触发（**cost 默认 0，不与养塔争木**）；被动技按养塔等级解锁（强化养成曲线）。
- **主动技触发（R6）**：①**手动**——选中塔 → 信息卡技能按钮（带 CD 环）→ 点放；②**自动**——设置 `auto_cast_active`(S22) 开启后，CD 好且有目标即自动释放（手动优先）。
- **被动生效**：达解锁等级后，在战斗事件（命中/击杀/状态）上常驻/触发（如破甲、冰封易伤、导电、腐蚀、逆风惩罚）。
- **技能清单（7 塔种）**：箭=精准齐射/破甲/连射；炮=饱和轰炸/燃烧溅射/重创；冰=极寒领域/霜冻蔓延/冰封易伤；**风=飓风之眼/持续气流/逆风惩罚**；魔=奥术爆发/奥术穿透/法力充能；毒=剧毒新星/腐蚀/毒性传染；电=雷暴/过载/导电。具体数值见 S28（全 `[PLACEHOLDER]`）。
- **设计红线**：无主导策略（7 塔互补）、不争木（主动技 cost=0）、认知可控（每塔 1 按钮 + 2 自动被动，无二级菜单深水区，P3）。
- **升级解锁（R3）**：被动①达 `passive1_lv`、被动②达 `passive2_lv`（全 `[PLACEHOLDER]`）；卖塔重建回 Lv1 重新锁定。
- **Dependencies**：Build（UI 入口）、Combat（效果结算 S5）、Economy（木不消耗）、Settings（auto_cast_active）、Save（不持久化）。

## 5.9 Mechanic: 玩家等级系统（Player Level System，新增 · S29）

> 详规见 **S29 玩家等级系统**；本条为 GDD 层权威定义。

- **Purpose**：P5 短局长线的元进度主轴——玩家等级 = 跨局持久化进度，是功能解锁的**统一门槛**，且每级提供**全局塔基础属性增益**（自动套到战斗内塔上，玩家无需操作）。
- **不累加语义（铁律）**：维护 `player_level_config[level] → bonus_attributes` 查表；战斗内塔有效属性 = `tower_base × player_level_config[当前等级].bonus`。**仅取当前等级那一行，绝不把 1..N 级加成求和**（非 `×Π` 亦非 `＋Σ`）。升级只换"取哪一行"，不叠加历史行。
- **等级→解锁（门槛）**：原 S11 资源解锁树改为由等级驱动——达到 `unlock_config[feature].required_level` 即解锁对应功能（塔种/永久升级/系统入口）；资源类解锁货币 TBD（见 S11/S29）。
- **XP 来源**：每次战斗结算(S08)产出 `xp_gain`（[PLACEHOLDER]），累加 `current_xp` → 跨过 `xp_required` 阈值即升级。
- **加成应用时机**：战斗开局/S02 建塔时，按 `session_player_level = save.player_level` 快照查 `player_level_config` 修正塔基础属性（dmg/range/atk_speed）；normal 流程中等级不变，故同局所有塔享用同一行加成。
- **设计红线**：无主导策略（等级增益普惠、不偏某塔种）；无认知过载（全自动、玩家零操作）；无支柱漂移（服务 P5 跨局变强留存）。
- **Dependencies**：S08（XP 来源）、S18（持久化）、S11（解锁门槛）、S02/S05（战斗加成引用）。

> 注：本 §5.9 顺延自 §5.8；GDD 中"胜负"实为 §5.7，而 SYSTEM_BREAKDOWN/FEATURE_SCOPE 多处引用为 §5.9（既有编号不一致，非本任务引入），建议后续统一编号或修正引用。

## 8. 从《绿色循环圈》到微信小游戏的适配与取舍

| 绿圈原味 | 微信小游戏适配 | 取舍说明 |
|---|---|---|
| PC 多键、多塔精细操作 | 单指点选 + 塔种轮盘 | 牺牲微操深度，换零门槛（P3） |
| 金/木手动频繁兑换 | "应急兑换"一键批量(受限) + 怪掉木自动入账 | 保留经济循环（P1），降操作负担 |
| 50 波长局、合作分工 | 3–5 分钟一局 + 元进度 | 砍单局时长保留存（P5），合作改为异步排行榜 |
| 极限养 / 极限漏高玩策略 | 保留"养塔"核心，弱化极限漏 | 聚焦 P2 养成爽感 |
| 魔免/空/重甲克制 | 保留克制但塔种精简到 7 类 | 保证 P4 多样性同时控复杂度 |

**不移植**：多人实时合作、复杂刷木微操（木已 session 化，无刷木）、超长 50+ 波硬核局。
