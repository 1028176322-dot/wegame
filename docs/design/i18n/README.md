# 统一文本配置表（i18n）

> 关联：GDD §5 / 全 29 系统策划案（`docs/design/systems/S01–S29_*.md`）
> 状态：v1 · 初版提取（覆盖全系统可见 UI 文案、提示、系统消息、异常兜底、技能名/描述）
> 说明：本表为游戏内**所有用户可见文本的唯一真理源**。代码**严禁硬编码字符串**，一律按 `id` 读取。

## 1. 文件

- `text_config.csv` — 文本主表（id / zh-CN / en / zh-TW / context）
  - `id`：唯一键，UPPER_SNAKE_CASE，命名空间前缀（见 §3），**永不暴露给玩家、永不改变**
  - `zh-CN`：中文真理源（默认语言）
  - `en` / `zh-TW`：**多语言脚手架，当前留空**，由本地化流程填充
  - `context`：用途/来源说明（如 `S03 应急兑换面板标题`），供策划/翻译定位

## 2. 加载契约（代码侧）

1. 构建/启动时加载 `text_config.csv` → 解析为 `Map<id, Record<lang, text>>`。
2. 运行时取文本：`getText(id, lang)`：
   - 取 `lang` 列对应值；
   - 若该语言缺失或为空 → **回退 `zh-CN`**（保证永不空文本）；
   - 若 `id` 不存在 → 返回 `id` 本身并记 S25 告警（便于发现漏配）。
3. 动态占位符：文本中 `{x}` `{y}` `{n}` `{m}` `{xp}` `{feature}` 等为格式化参数，由调用方按 `String.format` / 模板替换填入（如 `HUD_WAVE` 的当前/总波数）。
4. 任何 UI 文案变更 → **只改本表**（及对应语言列），不动代码字符串。

## 4. 多语言扩展

- **新增语言 = 新增一列**：在 `text_config.csv` 表头追加列（如 `ja`），逐行填译文本；`id` 与加载契约不变。
- 翻译由本地化流程填充 `en` / `zh-TW` / 其他列；**未填则自动回退 `zh-CN`**（见 §2.2）。
- 不建议为不同语言改 `id` 或新增副本文件——单表多列是唯一事实源。

## 5. 运行时落点（Cocos Creator 3.8.8）

- 设计副本（本文件）为真理源；工程侧将其置于 `assets/config/i18n/text_config.csv`（或 `resources` 下），由 `JsonUtil`/资源加载在启动时读入内存表。
- 构建管线可将 CSV 导出为代码可加载格式（如 `text_config.json`）以提升解析效率；导出物由本 CSV 生成，**不手动编辑**。
- 切换语言：`i18nManager.setLanguage(lang)` → 后续 `getText` 按新 `lang` 取值；HUD/面板监听语言变更事件刷新。

## 6. 维护约定

- **新增可见文本**：先在本表加一行（分配命名空间内唯一 `id`），再在 UI 代码引用 `id`；禁止直接写中文串。
- **删除/改名文本**：改 `zh-CN` 值即可；`id` 保持稳定（避免代码引用断裂）。
- **覆盖度**：本版为全系统初提取（含 E01–E12 异常兜底与 21 个技能名）。后续若发现遗漏文本，按 §3 命名空间补 `id` 即可；en/zh-TW 留待本地化。
