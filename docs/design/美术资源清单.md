## §3 规格与预算（权威：S19/F34 + SPEC_STANDARD §4）

- **格式**：PNG-8 / ETC2 / 合图集（Atlas）；微信小游戏单图 **≤128KB**，超则合图集。
- **压缩分包**：所有图经 `S19` 资源管线压缩分包（F34 美术 / F35 音频）。
- **DPR 分级**：美术出 **@1x / @2x / @3x** 三档；引擎 `cc.view.resolutionPolicy = FIT`，按 `devicePixelRatio` 选档；文本用引擎 **SDF 字体**。
- **相对比例**：UI 尺寸 = 基准 × `safe_scale`，`safe_scale = min(screen_w/750, screen_h/1334)`；状态图标 `24×24 × safe_scale`。
- **切片**：九宫（弹窗底 3×3 / 16px 边）、单格（图标）、序列帧等分（特效 8/10/12 等分）。
- **塔种主题配色**：箭炮=红、魔=紫、毒=绿、冰风=青（供飘字/状态 tint 统一，见 `systems/S30_attribute.md §4`）。
- **§4 表格模板**：见 `SPEC_STANDARD.md §4`（字段：资源名 / 帧数 / 分辨率 / 格式 / 切片）。

## §4 分包规划（S19_asset_subpackage）

- **主包**：启动背景、通用 UI 组件、大厅背景、核心塔/敌占位。
- **首分包**：各场景背景、HUD、经济、波次、图鉴、成就、签到、排行、赛季、设置、反馈、引导资源。
- **每关 art_pack**：`pkg_stage_<id>`（背景/敌皮/Boss 立绘/难度标识），进关前预载（见 `systems/S32_stage_config.md` `art_pack` 字段）。
- **对象池**：同屏大量弹道/粒子/状态图标须走对象池 + 上限（接 S23/S19），帧率保护。

## §7 机器可读骨架

配套 `art_resource_inventory.json`（同目录）提供 AI 直接解析的条目数组，字段：

```json
{
  "id": "bg_grass",
  "name": "地图背景底图（grass 主题）",
  "category": "background",
  "spec": { "resolution": "750x1334", "frames": 1, "format": "PNG", "slice": "nine-patch", "max_bytes": 204800 },
  "source_system": "systems/S01_map.md#4",
  "subpackage": "main",
  "reuse": "",
  "status": "planned",
  "priority": "P1"
}
```

> 该 JSON 为设计态清单（status=planned），后续接 art-pipeline 生产时由 `current_status`（generated_ok / dim_mismatch / missing / pending_3d）驱动执行。
