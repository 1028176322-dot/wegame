# SECURITY_RULES.md — 安全与 Git 规则（current）

> 依据方案 §15、§16。安全底线不因规模或原型阶段降级。

## 秘密（方案 §15.3）

- 密钥只能来自安全凭据系统或环境变量；禁止硬编码。
- 扫描器输出必须脱敏，只含规则ID、路径、行号、不可逆指纹。
- 当前工作区 0 命中不代表历史已清理；密钥轮换、工作区清理、Git 历史重写是三个独立动作。
- 历史重写属于 C3/P0，须协调所有开发者并单独授权。

## 文件操作（方案 §16.1）

删除/移动/重命名默认 dry-run。破坏性操作前执行四步门：hash 一致、全文引用为 0、构建/脚本未引用、建立可验证安全点。删除后重扫引用/构建/测试。

## Git 规则（方案 §16.2）

- 未经授权不 commit / push / force-push / 重写历史。
- 不使用破坏性回滚清除他人未提交改动。
- 混杂工作区按任务拆分审计，不整体提交。
- 提交对应一个共同根因或可独立回滚的交付；信息含 task ID、目的、验证、风险。
- 发布 tag / 版本号 / 变更日志由 Release Owner 批准。

## 编码与路径（方案 §16.3）

- 文本统一 UTF-8；禁止 U+FFFD、混合编码、不可解释 BOM 变化。
- 文档/脚本/配置使用仓库相对路径；运行时绝对路径只来自环境注入。
- 跨平台检查大小写、分隔符、保留名、最大长度、符号链接；`.gitattributes` 固定换行/二进制规则。

## 数据分类（方案 §15.7）

public / internal / confidential / restricted。当前项目默认 internal；引入用户数据/账号/支付前须补合规附录。
